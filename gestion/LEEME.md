## Metodología de gestión: Scrumban

Este proyecto usa **Scrumban** — una hibridación de Scrum y Kanban adaptada a equipos de investigación académica:

| Elemento Scrum | Elemento Kanban | Adaptación para investigación |
|---|---|---|
| Sprint de 4 semanas | Flujo continuo | Sprint = 1 mes calendario |
| Sprint Planning | Pull system | El equipo jala tareas cuando tiene capacidad |
| Daily standup | Límites WIP | Sincronía semanal asíncrona (no diaria) |
| Sprint Review | Visualización | Revisión mensual + demo de avances |
| Retrospectiva | Mejora continua | Cada 3 meses (trimestral) |

### Ceremonias

| Ceremonia | Frecuencia | Duración | Participantes |
|---|---|---|---|
| Sprint Planning | Primer lunes del mes | 2 h | Todo el equipo |
| Sincronía semanal | Todos los jueves (16:00) | 30 min | Todo el equipo |
| Sprint Review | Último viernes del mes | 1 h | Todo el equipo + invitados |
| Retrospectiva | Cada 3 meses | 90 min | Todo el equipo |

### Definición de Terminado (DoD)

- **Tarea de datos:** Dataset limpio en formato acordado, subido a GitHub con README.
- **Tarea de reporte:** PDF disponible en el sitio web del proyecto y compartido por correo con el equipo.
- **Tarea de código:** Código documentado, probado, en GitHub con README de reproducibilidad.
- **Tarea de artículo:** Manuscrito enviado a revista indexada (o carta de aceptación).
